Disabling Sample Applications

  • Disabling the deploy directory configuration of the sample applications

    The simplest way to disable the GraniteDS sample applications is to remove the reference of their custom deploy directory.

    Edit the <JBOSS_HOME>/server/default/conf/bootstrap/profile.xml file and comment out the ${jboss.server.home.url}samples and ${jboss.server.home.url}samples-ssl URIs:

    <property name="applicationURIs">
    <list elementClass="java.net.URI">
        <value>${jboss.server.home.url}deploy</value>
        <!--value>${jboss.server.home.url}samples</value-->
    </list>
</property>

    The GraniteDS sample applications won't be loaded anymore when you restart JBoss.

  • Deleting the deploy directory of the sample applications

    You may additionally delete the entire <JBOSS_HOME>/server/default/samples directory.
